*The* Buggy Editor

A practical buggy editor that avoids headaches.

My buggy editor's form has intuitive input methods: a colour picker for colours, drop down lists for limited options and integer only inputs for where you'd expect them.

Clean inputs - clean JSON

The carefulyl guided form means that you're less prone to getting undesired entries in the .json file for your buggy.

2-RULES Add validation according to the game rules

Intger based rules were set in HTML while comparative rules were set in Python.

  • Wheels and tyres must be even and greater than or equal to four: set HTML form type to integer and setting the min to 4 and the step by 2.
  • Select power types can only have one power unit: created a Python function to validate these two, via an if-else clause, and added that to the conditional while loop that validates inputs.
  • Integer value inputs have min values set in HTML.
  • Flag colours must not be the same, unless the pattern is plain: Python function that uses an if-else clause that is used in the same conditional while loop for validating inputs.
  • Race computer algorithm cannot be 'buggy': the option is not present for the user to click on in the drop down list.
